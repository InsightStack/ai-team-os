# Team

Team-level operating docs. Onboarding, rituals, and who's on the team.

## Doc index

- `roster.md`: who's on the team, roles, and primary tool handles.
- `onboarding.md`: what a new hire needs to know in their first two weeks.
- `rituals.md`: standing meetings, weekly rhythms, retros, and how the team operates.

## When to add a doc

Add a doc here when it describes how the team works rather than what the team is building. Examples:

- A new ritual or recurring meeting → `rituals.md` (or a new file if substantial).
- A new role or team-wide responsibility shift → update `roster.md`.
- A revised onboarding plan → `onboarding.md`.

If the content is about a customer, the product, or analytics, it lives elsewhere. See root `README.md` for the full map.

## Style

Plain language. Keep each file tight. If `rituals.md` or `onboarding.md` grows past 200 lines, split it.
